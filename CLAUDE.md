# CLAUDE.md

このリポジトリで作業するAI/開発者向けのガイド。編集前に必ず目を通すこと。

## プロジェクト概要

super2D — 1枚のイラストを投入するだけで、そのキャラがブラウザ上で
可愛く動き出す(呼吸・まばたき・髪の揺れ・アクション)ことを目指す
2Dキャラクターアニメーションツール。パーツ画像の生成はGemini API、
それ以外はすべてブラウザで完結するJavaScript(サーバー無し・ビルド無し)。
姉妹プロジェクト`ghostitd`(3D版)の設計・世界観を引き継いでいる。

## ドキュメントの読み方(重要)

- **設計の正は[`PLAN.md`](./PLAN.md)文末の「全体レビューによる設計改訂
  (2026-07-22)」節**。PLAN.mdは時系列の追記で成長してきたため、それ
  以前の節にはシート方式・パーツ別関節検出など**廃止済みの記述が残って
  いる**。矛盾を見つけたら常に改訂節が勝つ。廃止済み箇所には引用ブロック
  (`>`)で注記がある
- 実装タスクの分割と受け入れ条件は[`TASKS.md`](./TASKS.md)。**着手前に
  必ず該当タスクの受け入れ条件を読み、完了時にそれを満たしたことを確認
  する**
- キャラクター保存形式の仕様は`docs/character-format.md`(Stage Aで作成
  予定。存在するようになったら、フォーマット変更時は必ず同時更新する)

## 絶対に守るルール

1. **設計判断を変えない**。実装中に「こうした方が良いのでは」と思ったら、
   実装せず提案としてユーザーに確認する。スコープの拡大・縮小・迂回も
   同じ。過去に「勝手な判断でスコープを変えるな」という明確な指示が
   出ている
2. **Gemini APIキーをこのリポジトリ・環境に持ち込まない**。キーは
   ユーザーのブラウザのlocalStorageにのみ存在する設計。開発中の動作
   確認はプレースホルダー(円・四角の偽画像)と`fixtures/`(ユーザーが
   実生成して保存した実レスポンス)で行う。**開発セッションから実APIを
   呼ぶことは想定しない**
3. **UI/フロントの変更は必ずブラウザで実際に動かして確認してから完了と
   する**。この環境にはPlaywright+Chromiumがある
   (`executablePath: '/opt/pw-browsers/chromium'`、
   `NODE_PATH=/opt/node22/lib/node_modules`)。スクリーンショットを
   撮って目視し、console errorが無いことを確認する
4. **作業単位ごとにコミットし、`claude/new2d3d-latest-plan-dwzqlw`に
   プッシュする**。mainへのプッシュはユーザーの明示指示があった時のみ
   (fast-forwardで)
5. **エラーを黙って握りつぶさない**。catchして無視するくらいなら
   ユーザーに見えるところに生のメッセージを出す

## コード規約

- **ビルド無し・サーバー無し・素の`<script>`読み込み**(ESモジュール
  不使用)。GitHub Pagesの静的配信でそのまま動くこと
- 共有ロジックは`js/`配下に置き、`(function(global){ ... })(window)`で
  包んで**`window.S2D`名前空間**に公開する(ghostitdの`P3D`と同じ規約)
- 層の分離: API呼び出し層(fetch・キー管理)/データ変換層(切り抜き・
  QC・圧縮などの決定的処理)/UI層(タブ・プレビュー)。UI層からfetchの
  実装詳細が見えないようにする
- コメントは「なぜ」を書く(隠れた制約・過去の不具合の経緯)。「何を」は
  書かない。日本語でよい
- プロンプト文面はJSに埋め込まず`prompts/*.txt`を実行時にfetchする。
  `{ANGLE}`等のプレースホルダーは置換関数で埋める
- コア処理(正規化・合成・QC)は**ブラウザ実装が正**。`scripts/`配下の
  Nodeスクリプトは開発補助専用で、プロダクト実行時はNode非依存
- 物理・アニメーションは固定タイムステップ+描画側補間(再現性のため)

## テスト

`tests/`(整備中、TASKS.md B6参照)。整備後は「コード変更後は必ず実行、
全PASSを確認してからコミット」を慣習とする。それまでは上記ルール3の
Playwright手動確認を必ず行う。

## 動作確認の手順(例)

```bash
# 静的サーバーを起動(バックグラウンド)
nohup python3 -m http.server 8793 > /tmp/http.log 2>&1 &
# Playwrightで開いて確認(スクリーンショット+console error収集)
NODE_PATH=/opt/node22/lib/node_modules node <確認スクリプト>
```
